work flow
1. user signs up
2. user's credentials is checked for existing user
3. send err message if username already exists else save new user's credentials
4. redirect to login page
5. user enters credentials, if username does not exist send err message and send redirection link to signup page
6. if username exist but password does not match send err message and send redirection link to login page
7. if username exist and password match redirect to home page
8. redirect to menu page when user click's button to view full menu
9. add items to cart as user clicks add to cart button
10. calculate total amount as user clicks checkout button
11. when check out form is filled by user and pay button is clicked, process payment, send user credentials to database and send success message
